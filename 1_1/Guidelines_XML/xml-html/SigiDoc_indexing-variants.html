Letter variants and special characters

Elements

  • <c/>

Explanation

Important: c should be specified by a type attribute, whose value(s) listed here are the only being currently handled by SigiDoc/EFES autority lists and they should be used as they appear here: 

                    <c type="glyph" ana="glyphs.xml#Kappa.1" subtype="">[glyph here]</c>

Notice that here the attribute pointing to the authority list is ana (and not ref).

Designed for the diplomatic edition only and Athena Ruby as Greek font. The ana will populate the first column of the glyphs' index with the specific variants, tagged with c; the subtype's value should be:

1) in case of single letters, the corresponding standard Greek or Latin (or ...) letter of the referenced glyph;

1bis) in case of superscript Greek letters, the @subtype's value should be "superscript";

2) in case of ligatures, "greekLigatures" or "latinLigatures";

3) in case of decoration elements, "decoration";

4) in case of abbreviation signs, "abbreviation";

5) in case of editorial interventions, "editorial".

Not currently available.
