Dignities and Offices

Elements

  • <rs/>

Explanation

Important: rs should be specified by a type attribute, whose value(s) listed here are the only being currently handled by SigiDoc/EFES autority lists and they should be used as they appear here: 

                    <rs type="taxis" ref="...">[name here]</rs>

If a project does not want to differenciate dignities and offices, it could use this tag, with a general value "taxis" for the attribute type, in order to get a unique index in this matter (not currently handled by SigiDoc/EFES authoriy lists).

The spelling should be normalised in the authority list.
